# 任务模板

这个文件给使用者直接复制使用。

如果你不知道该怎么向 AI 描述需求，就从这里选一个最接近的模板。

## 使用原则

- 每次都先让 AI 阅读 `agent_protocol/PROTOCOL.md`
- 不确定改动落点时，让 AI 先判断
- 不会写技术方案没有关系，先把目标和结果说清楚

## 模板 1：先问问题，不改代码

适合：

- 我想先听建议
- 我不确定应该改哪里
- 我想先确认方案是否符合当前框架

```text
先阅读 agent_protocol/PROTOCOL.md。

我先不让你改代码。
请先基于这个仓库当前结构和协议，回答下面这个问题：

问题：
……

请回答时说明：
1. 这个问题主要涉及哪一层
2. 为什么
3. 如果要实施，推荐改哪些位置
4. 有没有和当前协议冲突的地方
```

## 模板 2：修改单个页面

适合：

- 调整页面布局
- 修改页面文案
- 修改页面交互
- 增加页面业务内容

```text
先阅读 agent_protocol/PROTOCOL.md。

我要修改一个页面。

目标：
……

位置：
我认为大概率在 apps/web，但如果你判断不对，请先纠正我。

我希望最终效果：
……

验收方式：
……

补充说明：
如果这是页面私有需求，请不要提前抽成共享能力。
```

## 模板 3：新增功能

适合：

- 新增一个完整业务功能
- 串起页面、接口、交互流程

```text
先阅读 agent_protocol/PROTOCOL.md。

我要新增一个功能。

功能目标：
……

用户会如何使用：
……

我已知会影响的位置：
……

我希望最终结果：
……

请先做这件事：
1. 判断正确落点
2. 说明会影响哪些层
3. 再开始实施
```

## 模板 4：按设计稿开发

适合：

- 我有截图
- 我有 Figma
- 我已经有明确视觉要求

```text
先阅读 agent_protocol/PROTOCOL.md。
再阅读 user_docs/design-input-spec.md。

我要按设计要求开发。

目标页面或功能：
……

设计资料：
……

我最在意的点：
……

请先检查我给的信息是否足够。
如果不够，请先指出缺什么，再开始实施。
```

## 模板 5：新增共享组件或共享能力

适合：

- 多个页面都要复用
- 不希望在业务页面里复制实现

```text
先阅读 agent_protocol/PROTOCOL.md。

我要新增或修改共享能力。

目标：
……

为什么它应该是共享能力：
……

我预计会复用到这些地方：
……

请先判断它应该落在：
- packages/ui-components
- packages/ui-theme
- packages/services
- 或其他更合适的位置
```

## 模板 6：修复 Bug

适合：

- 页面异常
- 样式错乱
- 接口行为不对
- 交互和预期不一致

```text
先阅读 agent_protocol/PROTOCOL.md。

我要修一个 Bug。

问题现象：
……

出现位置：
……

复现步骤：
1. …
2. …
3. …

我预期的正确结果：
……

请先分析问题属于页面、共享组件、主题还是服务层，再修复。
```

## 模板 7：优先确认本地服务端联调

适合：

- 我希望前端直接对接本地后端
- 我想先确认环境变量和服务是否齐全
- 我不希望 AI 使用 mock 作为默认方案

```text
先阅读 agent_protocol/PROTOCOL.md。

这次请使用本地服务端联调，不使用 mock。

需求目标：
……

请先说明：
1. 应该运行什么命令
2. 需要哪些本地服务
3. 当前还缺哪些 `.env.development` proxy 配置
4. 哪些接口需要先由后端提供
```

## 如果你完全不会判断

直接用这个最简单版本：

```text
先阅读 agent_protocol/PROTOCOL.md。

我不会判断这个需求应该改哪里。
请你先根据当前框架判断正确落点，再回答我是否需要改代码，以及应该怎么改。

我的需求是：
……
```
