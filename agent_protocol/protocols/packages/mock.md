# @workspace/mock 协议

## 宪章

`@workspace/mock` 是本仓库的本地开发与验证 mock 层。

它只负责模拟后端接口与外部资源行为，帮助应用在没有真实后端时继续开发、联调和演示。

## 强制规则

- mock 必须尽量保持与真实接口契约一致，包括路径、返回结构和关键字段。
- mock 只能模拟服务行为，禁止承载页面、路由、组件或业务 UI 实现。
- mock 数据应优先服务本地开发、演示和前端联调，不应替代真实服务契约设计。
- 新增 mock 接口时，必须优先对齐 `@workspace/services` 中现有 API 定义，禁止另起一套字段命名。
- 若真实接口发生变化，mock 应同步更新，避免前端只在 mock 环境可用。

## 边界

- 可以放：
  - MSW handlers
  - mock session / mock 文件记录 / mock 响应工厂
  - 浏览器端 mock 启动入口
- 不应放：
  - 页面逻辑
  - UI 组件
  - 通用请求基础设施
  - 与单一页面强绑定且不可复用的临时代码

## 完成定义

- mock 模式可以被应用正常启用
- 关键请求路径已被拦截
- 返回结构与真实 API 契约一致
- 不污染真实环境和真实请求流程
